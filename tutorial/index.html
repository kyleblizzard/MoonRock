Learning C by Building a Compositor

A beginner's guide to C programming, using the Crystal Compositor from AuraOS as our running example

Most programming tutorials teach you C with toy examples. Calculate a tip. Sort a list. Yawn. We're going to do something different. We're going to learn C by looking at a real program that does something genuinely cool: Crystal Compositor, the piece of software that controls every single pixel on your screen in AuraOS. Every window, every shadow, every animation — Crystal draws it all using OpenGL and about two thousand lines of C.

You don't need any programming experience to follow along. We start from zero, build up the fundamentals, and at every step I'll show you how those fundamentals appear in real compositor code. By the end, you won't just understand C — you'll understand how a compositor works, which is one of the most satisfying things a programmer can learn.

What You'll Need

Not much, honestly. If you're reading this, you're already most of the way there.

  • A computer. Mac, Linux, Windows — doesn't matter. C runs everywhere. That's kind of the whole point.
  • A text editor. VS Code is great. Sublime Text is great. Vim if you're feeling brave. Notepad if you're feeling masochistic. Whatever you're comfortable with.
  • A terminal. That black window with the blinking cursor. On Mac it's called Terminal. On Linux you probably already have one open. On Windows, install WSL and use that.
  • A C compiler. On Mac: xcode-select --install. On Linux: sudo apt install gcc. On WSL: same as Linux.
  • Curiosity. This is the most important one. If you want to know how things work — really work, at the metal — you're in the right place.
A note on the Crystal code: Don't worry if the real code examples look intimidating at first. That's completely normal. The whole point of this tutorial is to get you from "what does any of this mean" to "oh, I see exactly what's happening here." We'll take it one piece at a time.
