Chapter 2: Your First Program

The Sacred Tradition

Every programming tutorial since the dawn of time starts with "Hello, World." It's tradition. It's comforting. It's also surprisingly useful, because if you can get this tiny program to compile and run, you've proven that your entire toolchain works — the editor, the compiler, the terminal, all of it. That's not nothing.

Open your text editor. Create a new file called hello.c. Type this: 

#include <stdio.h>

int main(void)
{
    printf("Hello, World!\n");
    return 0;
}

That's it. Six lines. Let's break down every single one.

Line by Line

#include <stdio.h>

This is a preprocessor directive. Before the compiler even looks at your code, a preprocessor runs through it first and handles lines that start with #.

#include means "copy the contents of this file into my program." It's like importing a toolbox. stdio.h stands for "standard input/output header" — it gives you access to functions like printf that let your program talk to the screen.

Without this line, the compiler wouldn't know what printf is. It would look at your code, see printf, and say "I have no idea what that means" and refuse to compile. Fair enough.

int main(void)

This is the entry point of your program. When you run your program, the computer looks for a function called main and starts executing the code inside it. Think of it as the starting line of a race. Every C program has exactly one main.

int means this function returns an integer (a whole number) when it's done. void in the parentheses means it takes no inputs. We'll talk more about functions in Chapter 4.

{ } — The Curly Braces

Everything between { and } is the body of the function. It's the actual instructions. "When main runs, do everything between these braces."

printf("Hello, World!\n");

This is the money line. printf stands for "print formatted" — it sends text to the terminal. The text you want to print goes inside double quotes. The \n at the end is a newline character — it tells the terminal to move to the next line after printing, so your prompt doesn't awkwardly land on the same line as your output.

The semicolon at the end is how C knows you're done with a statement. It's like a period at the end of a sentence. Forget it and the compiler will complain. You will forget it. Everyone does. It's a rite of passage.

return 0;

This tells the operating system "I'm done, and everything went fine." A return value of 0 means success. Any other number means something went wrong. This is a convention that goes all the way back to Unix — and every script, every process manager, every CI pipeline in the world still checks this number.

Compile and Run

Save your file. Open a terminal. Navigate to wherever you saved hello.c, and type: 

gcc hello.c -o hello

Here's what just happened:

  • gcc — the GNU C Compiler. It reads your C code and turns it into a program your computer can actually execute.
  • hello.c — your source file. The code you wrote.
  • -o hello — "name the output file hello." Without this, gcc would name it a.out, which isn't very helpful.

If there are no errors (and there shouldn't be, unless you made a typo), you'll get no output. Silence is golden in the terminal — it means everything worked.

Now run it: 

./hello

And you should see: 

Hello, World!

Congratulations. You just told a computer what to do and it listened. That might not seem like much, but every program ever written — every game, every operating system, every compositor that draws windows on your screen — started right here. Someone typed some text into a file, compiled it, and ran it. The difference between this and Crystal Compositor is just scale and knowledge. The process is exactly the same.

What a Real main() Looks Like

Our hello world has a two-line main. Crystal Compositor's main is a bit more involved. Here's the actual entry point of the AuraOS window manager: 

#include "wm.h"
#include "events.h"
#include "crystal.h"
#include <stdio.h>
#include <signal.h>
#include <stdlib.h>

static AuraWM wm;

static void handle_signal(int sig)
{
    (void)sig;
    fprintf(stderr, "[aura-wm] Received signal %d, shutting down\n", sig);
    wm.running = false;
}

int main(int argc, char *argv[])
{
    (void)argc;
    (void)argv;

    fprintf(stderr, "╔══════════════════════════════════════╗\n");
    fprintf(stderr, "║         AuraOS Window Manager        ║\n");
    fprintf(stderr, "║   Pixel-Perfect Snow Leopard in C    ║\n");
    fprintf(stderr, "╚══════════════════════════════════════╝\n");

    // Handle shutdown signals gracefully
    signal(SIGINT, handle_signal);
    signal(SIGTERM, handle_signal);

    // Initialize the window manager
    if (!wm_init(&wm, getenv("DISPLAY"))) {
        return 1;
    }

    // Initialize Crystal compositor for shadows and effects
    if (!crystal_init(&wm)) {
        fprintf(stderr, "[aura-wm] Crystal unavailable — no shadows\n");
    }

    // Run the event loop (blocks until wm.running = false)
    events_run(&wm);

    // Clean shutdown
    crystal_shutdown(&wm);
    wm_shutdown(&wm);

    return 0;
}

Don't panic. You're not supposed to understand all of this yet. But notice the shape of it. See how it's still just main with some function calls inside? It includes some headers, sets up some things, runs a loop, cleans up, and returns. The structure is the same as our hello world — it's just doing more.

A few things to notice:

  • There are more #include lines — it's pulling in multiple toolboxes because it needs more tools.
  • fprintf(stderr, ...) is like printf, but it writes to the error stream instead of the normal output. That's where log messages go.
  • The signal() calls set up handlers so the program shuts down cleanly when you press Ctrl+C.
  • return 1; on failure — remember, non-zero means "something went wrong."
  • The program initializes things in order, runs a loop, then shuts down in reverse order. That pattern — set up, run, tear down — is everywhere in systems programming.
The reverse-order shutdown is worth remembering. Crystal depends on the window manager being initialized, so Crystal must be shut down before the window manager. Last in, first out — like a stack of plates. You'll see this pattern for the rest of your programming career.

Try It Yourself

Modify your hello.c to print your name. Then try printing two lines. Then try printing a box like Crystal's banner using regular characters: 

+------+
| Kyle |
+------+

Each printf call prints one line. You'll need several of them.
