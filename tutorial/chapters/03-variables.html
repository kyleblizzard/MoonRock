Chapter 3: Variables — Giving Names to Things

Boxes with Labels

Imagine you have a wall of mailboxes. Each one has a label on it and can hold one thing. That's basically what a variable is — a named container that holds a value. You put something in, you can read it later, you can change it, you can use it in calculations. That's the whole idea. 

int age = 28;           // a whole number
float pi = 3.14159;     // a decimal number
char letter = 'A';      // a single character

Three variables, three different types. The type tells the computer what kind of data the box can hold and how much memory to set aside for it. An int takes 4 bytes. A char takes 1 byte. A float takes 4 bytes but can store decimals.

Types are like labeling your boxes. You wouldn't put soup in a shoebox. You wouldn't store a novel in a thimble. The type tells C what to expect, so it can store the value efficiently and catch mistakes when you try to do something that doesn't make sense.

The Core Types

C has a small set of built-in types. You'll use these constantly:

  • int — Whole numbers. 42, -7, 0. Your workhorse type for counting, indexing, and general-purpose integer math.
  • float — Decimal numbers. 3.14, 0.001, -99.5. When you need fractions.
  • double — Like float but with more precision (8 bytes instead of 4). Use this when accuracy matters — scientific calculations, graphics math.
  • char — A single character. 'A', 'z', '7'. Under the hood it's just a small number (0-255) that maps to a character via ASCII.
  • bool — True or false. Yes or no. On or off. You need #include <stdbool.h> to use it, which gives you true and false.
Why does bool need an include? Original C (1972) didn't have a boolean type — people just used int and treated 0 as false and anything else as true. The bool type was added later in C99, so you need the header to access it. Old habits die hard in C.

Declaration vs. Assignment

You can declare a variable (create the box) and assign it (put something in) separately: 

int score;           // declaration — creates the box (value is garbage!)
score = 100;         // assignment — puts 100 in the box

Or you can do both at once, which is what most people do: 

int score = 100;     // declaration + assignment in one line
Uninitialized variables are dangerous. If you declare a variable without assigning a value, it contains whatever random garbage was already in that memory. It could be 0, it could be 847293, it could be anything. Always initialize your variables. This is one of C's classic footguns.

Printing Variables with printf

printf uses format specifiers — little placeholders that tell it what type of value to print and where to put it. The specifier starts with %: 

int x = 42;
float y = 3.14;
char c = 'K';

printf("x is %d\n", x);       // %d for integers
printf("y is %f\n", y);       // %f for floats
printf("c is %c\n", c);       // %c for characters
printf("x=%d, y=%.2f\n", x, y);  // multiple values, .2 = 2 decimal places

The %d gets replaced by the value of x. The %f gets replaced by the value of y. You can have as many format specifiers as you want — just make sure you pass the right number of values after the format string, in the right order. If you mix them up, C won't stop you, and you'll get weird results. Ask me how I know.

Constants: Values That Never Change

Sometimes you have a value that should never be modified. A shadow radius. The value of pi. The maximum number of windows your compositor can track. For these, you use a constant.

In C, there are two common ways to make constants. The first is #define, which is a preprocessor directive — it does a find-and-replace across your code before compilation: 

#define SHADOW_RADIUS  22
#define MAX_CLIENTS   256

Everywhere the compiler sees SHADOW_RADIUS, it substitutes 22. It's like a text macro. The convention is ALL_CAPS for #define constants, so they stand out in your code.

The second way is the const keyword: 

const int max_speed = 120;   // can't be changed after this line

Crystal's Constants: A Real Example

Let's look at how Crystal Compositor uses named constants. Here are the shadow parameters from crystal.h: 

// Shadow parameters from Crystal Compositor

#define SHADOW_RADIUS             22
#define SHADOW_Y_OFFSET            4

// How far the shadow extends past the window on each side
#define SHADOW_TOP     (SHADOW_RADIUS - SHADOW_Y_OFFSET)   // 18 px
#define SHADOW_BOTTOM  (SHADOW_RADIUS + SHADOW_Y_OFFSET)   // 26 px
#define SHADOW_LEFT     SHADOW_RADIUS                      // 22 px
#define SHADOW_RIGHT    SHADOW_RADIUS                      // 22 px

// Shadow opacity: 0.0 = invisible, 1.0 = fully opaque black
#define SHADOW_ALPHA_ACTIVE    0.45
#define SHADOW_ALPHA_INACTIVE  0.22

Look at how readable this is. SHADOW_RADIUS is 22 pixels — that's how far the Gaussian blur spreads. SHADOW_Y_OFFSET is 4 pixels — the shadow shifts down slightly, like light coming from above. It's saying: "whenever I write SHADOW_RADIUS, I mean the number 22."

Notice how SHADOW_TOP is defined in terms of the other constants: (SHADOW_RADIUS - SHADOW_Y_OFFSET). That evaluates to 18. If someone later changes SHADOW_RADIUS from 22 to 30, all the dependent values update automatically. That's the power of named constants — change one number and everything stays consistent.

And look at the alpha values: the focused window gets 0.45 opacity (a strong, prominent shadow) while inactive windows get 0.22 (softer, so they recede visually). These numbers were probably tweaked dozens of times until the shadows looked just right. Having them as named constants makes that iteration painless — you change one number, recompile, and see the result.

The window manager also defines its maximum capacity this way: 

#define MAX_CLIENTS  256    // Maximum number of managed windows

256 windows. That's the limit. If you tried to open a 257th, the window manager wouldn't track it. (That's a simplification — in practice, who has 256 windows open? If you do, you have bigger problems than your window manager.)

Variables Can Change

Unlike constants, regular variables can be updated. That's kind of the point: 

int score = 0;
printf("Score: %d\n", score);   // Score: 0

score = score + 10;
printf("Score: %d\n", score);   // Score: 10

score += 5;                      // shorthand for score = score + 5
printf("Score: %d\n", score);   // Score: 15

The += operator is shorthand that saves you from typing the variable name twice. C also has -=, *=, /=, and a few others. They all work the same way: take the current value, do the operation, store the result back.

Try It Yourself

Write a program that:

  1. Declares an int for your birth year.
  2. Declares an int for the current year (2026).
  3. Calculates and prints your age.
  4. Declares a float for your height in meters and prints it with 2 decimal places.
  5. Defines a #define constant for the maximum age a human can reach, and prints it.

Hint: printf("Age: %d\n", current_year - birth_year);
